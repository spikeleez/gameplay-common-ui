Detailed Description

Manages the registry of gameplay settings for a local player.

This abstract class serves as the foundation for gameplay settings registries. It maintains a collection of settings, provides functionality for initialization, querying, and filtering settings, and handles setting-related events.

The registry keeps track of both top-level settings and all registered settings, allowing for hierarchical organization of settings. It also manages the relationship between settings and the owning local player.

Note
This is an abstract class that must be inherited by specific implementations.

◆ GetRegistry()

template<typename RegistryClass>
RegistryClass * UGameplaySettingRegistry::GetRegistry ( const ULocalPlayer * InLocalPlayer)
inlinestatic

Gets or creates a settings registry instance for a local player.

This template function retrieves an existing registry instance or creates a new one if it doesn't exist. The registry is stored as a subobject of the local player.

Template Parameters
RegistryClassThe type of registry to retrieve (must derive from UGameplaySettingRegistry)
Parameters
InLocalPlayerThe local player that owns or will own the registry
Returns
Pointer to the registry instance, or nullptr if InLocalPlayer is invalid
Note
The registry uses the class name as its unique identifier
If a new registry is created, Initialize() is called automatically

Example usage:

UCustomSettingRegistry* Registry = UGameplaySettingRegistry::GetRegistry<UCustomSettingRegistry>(LocalPlayer);
